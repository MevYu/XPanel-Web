# design-sync notes — XPanel-Web

Repo-specific gotchas for syncing this design system to claude.ai/design.

## Shape & build

- **Not a published library** — XPanel-Web is a Vite SPA. No `module`/`main`/`exports`
  in package.json, no shipped `.d.ts`. The converter runs in **synth-entry mode**
  (builds `_ds_bundle.js` from `src/` directly). `shape: "package"`.
- `srcDir: "src/components"` scopes component **discovery** to the component suite
  (otherwise synth mode would pull in pages/, layout/, auth/, App, etc.). Bundling
  still follows relative imports out of that dir, so components keep working.
- Components are styled with **Tailwind v4 utility classes** generated at build time.
  The source `src/styles/global.css` only has `@theme` tokens — it does NOT contain
  the utility classes. So `cssEntry` points at the **compiled** stylesheet from the
  app build (has `:root` token vars + all utilities + `@font-face` for Inter /
  JetBrains Mono / Space Grotesk, self-hosted via @fontsource).
- **Font URLs must be made relative.** The Vite build emits font urls as absolute
  `/assets/*.woff2`, which the converter can't resolve (it resolves relative to the
  stylesheet dir), so every `@font-face` dangles and designs fall back to system
  fonts. `.design-sync/prep-css.sh` strips the `/assets/` prefix and writes the
  stable `dist/assets/index.ds.css` that `cfg.cssEntry` points at. **Always run
  `bash .design-sync/prep-css.sh` after `npm run build`, before the converter.**

## Re-sync sequence (the build prep)

1. `npm run build` (regenerates `dist/`; git-ignored, so required on fresh clone)
2. `bash .design-sync/prep-css.sh` (writes `dist/assets/index.ds.css` — the stable
   `cssEntry`; absorbs the per-build hash change in the source filename)
3. run the converter / `resync.mjs`

## Discovery & previews (how this DS is wired)

- **Entry barrel:** repo is an app with no library entry, so `cfg.entry =
  .design-sync/entry.tsx` (re-exports the 16 components → `window.XPanel.*`) and
  `--entry` makes the converter resolve PKG_DIR to the repo root. Discovery is driven
  explicitly by `cfg.componentSrcMap` (16 entries). To add/remove a component, edit BOTH
  entry.tsx and componentSrcMap.
- **No provider needed** — components are context-free Tailwind. But the DS is
  **dark-theme** and the preview card body is white, so light text is invisible. Previews
  wrap content in a shared `.design-sync/_frame.tsx` (`Frame`) that paints
  `var(--color-bg)` behind the story. `_frame.tsx` lives OUTSIDE `previews/` (a file in
  `previews/` not matching a component name trips a harmless "stale preview" warning).
  Do NOT use `cfg.provider` for this — it would tell the design agent components need a
  context they don't.
- **Overlays (Modal, TabModal):** `cardMode: single` + viewport, AND the preview wraps the
  modal in a `transform: translateZ(0)` container so the `fixed inset-0` overlay resolves
  to the sized box instead of collapsing the captured card to blank (`[RENDER_BLANK]`).
- **Grid-wide rows (Card, Sparkline, Stat):** `cardMode: column` (else `[GRID_OVERFLOW]`).
- **FileTreeSidebar** preview supplies a mock `listDir` returning a resolved Promise of
  `DirEntry[]`; **CodeEditor** lazy-loads CodeMirror and renders fine in capture.
- Changing a `cfg.overrides` cardMode requires a full `package-build.mjs` — `preview-rebuild.mjs`
  refuses with `[CONFIG_STALE]`.

## Known render warns

None — 16/16 render cleanly, 0 floor cards. Any warn on re-sync is new; investigate it.

## Re-sync risks (watch-list)

- `cssEntry` (`dist/assets/index.ds.css`) is regenerated by prep-css.sh from a hashed
  source file; both `dist/` and the generated file are git-ignored. Forgetting prep-css.sh
  → stale/missing CSS or dangling fonts.
- Fonts re-upload deterministically; if `@fontsource` versions bump, the woff2 hashes in
  the compiled CSS change — re-run the full pipeline, don't hand-edit font paths.
- Previews import realistic ops-panel content (domains, paths) inline — purely composition,
  safe to edit. They track the component APIs in `src/components/`; if a component's props
  change, update its `previews/<Name>.tsx`.
- Heavy components (CodeEditor/CodeEditorView use CodeMirror; anything xterm) may not
  render statically — expected to fall back to floor cards.
